## 阅读说明

- **【前置条件】** = 必须满足才能触发的 GameState 条件
- **【触发】** = 执行后写入 GameState 的 Fact 或 Hook
- **【碎片】** = 触发 RUNE 收集 + 赵启叙事片段全屏覆盖
- **⚠️** = 常见卡点或设计注意事项

## 零、游戏入口

**Warning 页面**

玩家看到内容警示，选择：
- "我已了解，进入游戏" → `gentleMode = false` → 进入序章
- "开启柔和模式" → `gentleMode = true` → 进入序章（闪烁效果被抑制）

**Prologue 序章**

打字机叙事四段，播完后显示"打开笔记本电脑"按钮。
- 任何时候点"跳过序章" → 直接进入桌面
- 等待播完点按钮 → 进入桌面

⚠️ 序章现在的邮件引用是模糊版本（"我留了后门"），不剧透"她没有出院"。玩家此时只知道：妹妹失踪了，有人发来了一封奇怪的定时邮件。

### 1.7 林晓信号强度系统

`linXiaoSignalStrength = collectedRunes.length + signalBoost`

| 强度 | 视觉表现 |
|------|---------|
| 0-1 | 正常 |
| 2+ | 随机浮现幽灵文字（每8-15秒一次） |
| 4+ | 偶发全屏"……帮帮我……"白字闪烁 |
| 6+ | 红色渐变光晕覆盖层，有脉冲动画 |
| 7+ | 右下角出现"LX-044-YIN 信号已锁定" |

幽灵文字来自 AMBIENT_HINTS，均为情绪性词汇，不含剧透。

## 八、已知设计空洞（需修复）

**空洞一：`car_crash_clue` 无触发点**
`OA.tsx` 人事档案里的赵启注释需要 `hasFact('car_crash_clue')` 才显示，但这个 Fact 在代码里没有任何地方被 `addFact` 赋值。新闻页N1（2023-11-15报道）里有"作者：陈维远"但没有触发。
**建议修复：** 在新闻页N1阅读完整内容时 `addFact('car_crash_clue')`，或者直接移除这个条件判断，始终显示赵启的注释。

**空洞二：B2报告验证码 `mnt8023_zq` 的发现路径不明确**
玩家怎么知道要输入这个验证码？代码里没有任何地方明确展示或暗示这个字符串。
**建议修复：** 在维护日志某一条目里由赵启提到"我的个人验证码"，或者在影子档案README里给出提示。

**空洞三：OA地址 `oa.tranquil-sleep.com` 的发现路径**
`oa_url_discovered` 这个 Fact 目前有两个地方会被写入：FAQ论坛URL悬浮（这写的是论坛地址不是OA地址，存在逻辑混淆）和论坛管理员登录成功时。实际上OA的URL应该从管理员内部通讯里自然发现，目前代码里内部通讯内容没有明确包含OA URL字符串。
**建议修复：** 在管理员内部通讯某封邮件里自然提及 `oa.tranquil-sleep.com`，阅读时触发 `addFact('oa_url_discovered')`。

**空洞四：论坛桌面图标破坏发现感**
桌面左侧图标有"论坛"直通快捷方式，绕过了FAQ里悬浮发现URL的整个路径。
**建议修复：** 删除桌面"论坛"图标。
