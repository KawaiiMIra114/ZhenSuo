## 零、全局规则

- 所有叙事文字必须有打字机效果，逐字输出
- 打字机速度：正文每字40ms，代码/等宽字体每字25ms，PROJECT DELTA段落每字60ms
- 每个自然段之间停顿800ms，除非另有标注
- 结局动画播放期间禁止所有用户输入
- 结局A和C结束后，桌面执行"痕迹抹除"状态，以下变化静默执行，不显示任何提示：
  - 林晓微信联系人消失
  - 相册文件夹存在但内容为空
  - 爸爸聊天记录替换为新模板（内容见下）
  - 官网打开显示404
  - 论坛打开显示404
  - 关机按钮初始不亮，幽灵文字阶段结束后亮起
  - 林晓信号强度幽灵文字停止出现

## 五、刷新行为

页面刷新时清除所有localStorage存档，重新开始游戏（从Warning页面开始）。

实现方式：在入口文件（`main.tsx` 或 `App.tsx`）里，监听 `beforeunload` 或在初始化时判断是否为刷新（`performance.navigation.type === 1`），若是则调用 `localStorage.clear()`。

## 六、新增GameState字段

需要在 `GameContext.tsx` 里新增：

```typescript
// 游玩时长（秒）
playTimeSeconds: number

// 已完成结局列表（用于致谢页面按钮状态）
completedEndings: string[]  // ['ending_a', 'ending_b', 'ending_c']

// 痕迹抹除状态（结局A/C后永久生效）
erasureActive: boolean
```

`playTimeSeconds` 用 `setInterval` 每秒+1，存入localStorage。

`erasureActive` 为true时，Desktop渲染被抹除后的状态（联系人消失、照片清空等）。

## 七、注意事项

1. `window.open('/credits', '_blank')` 在部分浏览器会被弹窗拦截器阻止。需要确保调用发生在用户交互事件的同步上下文里（比如玩家点击关机按钮的那个事件链里），而不是在setTimeout或Promise的异步回调里。结局B的致谢弹出时机在动画序列结束后，需要用一个用户可见的"继续"按钮触发，而不是自动弹出。

2. 结局回放功能（致谢页面里的按钮）需要一个独立的结局直播模式，跳过所有游戏流程，直接渲染对应结局的动画组件。建议新增一个 `EndingReplay` 路由（`/credits?replay=ending_a`），在致谢页面里通过URL参数控制。

3. 刷新清档的实现需要区分"用户主动刷新"和"首次加载"，避免第一次打开游戏就清掉空存档触发异常。建议用sessionStorage做标记：首次加载写入标记，刷新时检测标记存在则清档。
