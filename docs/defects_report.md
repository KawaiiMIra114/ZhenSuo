## 二、Bug 级缺陷（P0）

### BUG-001：「柔和模式」按钮没有实际功能

| 项目 | 内容 |
|------|------|
| **文件** | `src/pages/Warning.tsx` L28-L33 |
| **描述** | 两个按钮（"我已了解"和"柔和模式"）执行完全相同的操作 `setCurrentApp('desktop')`。柔和模式没有任何状态标记，后续恐怖元素无法根据此设置进行削弱。 |
| **影响** | 承诺的功能不存在，影响玩家信任 |
| **修复方案** | 在 `GameContext` 增加 `gentleMode: boolean`，后续 Terminal/OA 据此控制恐怖效果强度 |

---

### BUG-002：没有存档 / 进度持久化

| 项目 | 内容 |
|------|------|
| **文件** | `src/GameContext.tsx` |
| **描述** | 所有游戏状态仅存在于 React state 中。刷新浏览器或关闭标签页后全部丢失，玩家必须从头开始。 |
| **影响** | 游戏完全不可中断，严重影响可玩性 |
| **修复方案** | 使用 `localStorage` 存储 `{ currentApp, clues, fragments }`，初始化时读取，变更时同步。结局页"重启系统"按钮需 `localStorage.clear()` |

---

### BUG-003：笔记本不显示「当前目标」

| 项目 | 内容 |
|------|------|
| **文件** | `src/components/Notebook.tsx` |
| **描述** | 大纲中规划的三级提示系统（卡关5/10/15分钟后的渐进提示）完全没有实现。笔记本中也缺少"当前目标"显示，玩家不知道下一步该做什么。 |
| **影响** | 玩家容易迷路放弃 |
| **修复方案** | 在 `GameContext` 增加 `currentObjective`，每收集关键线索时更新目标；增加计时器弹出"匿名线人邮件"提示 |

---

### BUG-004：无效的服务端依赖膨胀包体积

| 项目 | 内容 |
|------|------|
| **文件** | `package.json` |
| **描述** | `express`、`better-sqlite3`、`dotenv` 是纯服务端依赖，对纯前端项目完全无用 |
| **影响** | 安装时间延长、CI 可能因缺少编译工具链而失败 |
| **修复方案** | 移除 `express`、`better-sqlite3`、`dotenv` 及 `@types/express` |

## 三、游戏流程缺陷（P1）

### FLOW-001：第一幕（桌面）到第二幕（官网）的过渡太突兀

| 项目 | 内容 |
|------|------|
| **文件** | `src/pages/Desktop.tsx` |
| **描述** | 桌面上有邮件、诊疗单、报案回执三个文件和一个浏览器图标。但玩家看完三个文件后，**缺少明确的引导**告诉他们应该打开浏览器去访问诊所官网。诊所官网URL只出现在诊疗单翻转后的文字中，很容易被忽略。 |
| **影响** | 玩家可能在桌面阶段就卡住 |
| **修复方案** | 当三个文件都被阅读后，自动在桌面弹出一个"你在浏览器收藏夹中发现了一个地址"的通知，引导玩家点击浏览器图标 |

---

### FLOW-002：第二幕中的线索推理链断裂

| 项目 | 内容 |
|------|------|
| **涉及文件** | `Clinic.tsx`、`Forum.tsx` |
| **描述** | 当前设计要求玩家按以下链条推理：`诊疗单背面条码 → 挂号查询 → 3月14日新闻 → 机房 + 工号8023 → 论坛帖子 → "浮生无量"口令 → 赵启离职照片 → OA地址`。但这条链中有几个断点：|
| | 1. 新闻搜索只支持**精确包含**匹配（`.includes()`），玩家搜"林晓"找不到（新闻里写的是"林某某"） |
| | 2. 挂号条码 `LX-044-YIN` 没有任何线索指向它（诊疗单背面只写了刻字，没提到条码格式） |
| | 3. 论坛和官网的探索顺序完全线性，无法并行推进 |
| **影响** | 核心推理体验断裂，玩家可能需要穷举才能推进 |
| **修复方案** | ① 搜索改为模糊匹配或增加同义词映射；② 在桌面邮件中补充条码线索；③ 增加交叉引用（新闻提到论坛讨论、论坛帖子引用新闻） |

---

### FLOW-003：OA 密码推理缺乏反馈

| 项目 | 内容 |
|------|------|
| **文件** | `src/pages/OA.tsx` L19 |
| **描述** | 玩家需要从论坛帖子得知"浮生无量"口令，再从新闻得知"密码是个人口令拼音首字母"，最终推导出密码是 `fswltz`。**但如果玩家没有同时获得这两条线索，就完全无法登录OA**，且系统只显示冰冷的"ACCESS DENIED"，没有提供任何方向性提示。 |
| **影响** | 玩家在OA登录界面死磕，无渐进反馈 |
| **修复方案** | 增加智能提示：输入错误3次后显示"提示：密码规则见安全通知"；输入工号正确但密码错误时显示"工号已验证，密码错误" |

---

### FLOW-004：第三幕覆盖码的推理路径不够清晰

| 项目 | 内容 |
|------|------|
| **文件** | `src/pages/OA.tsx`、`src/pages/Clinic.tsx` |
| **描述** | 覆盖码 `taiyijiuku` 由两半组成：前半段来自林医生专栏文章的藏头诗（太乙救苦），后半段来自OA日志中符纸下隐藏的"JiuKu"。问题在于：|
| | 1. "整理文章标题"按钮太隐蔽（灰色小字放在页面底部） |
| | 2. 赵启日志里说"一半交给了林医生，让她**藏在专栏文章里**"——但玩家需要倒回官网才能找到，没有快捷入口 |
| | 3. 两半合并成 `taiyijiuku` 的拼接逻辑没有任何提示（是直接拼还是大小写？有空格吗？） |
| **影响** | 最后的高潮解谜体验被繁琐的猜测破坏 |
| **修复方案** | ① 赵启日志提供明确线索"把两个词拼在一起就是完整的码"；② "整理标题"按钮改为更明显的交互；③ 当两条线索都收集后在笔记本中自动显示"尝试将两段合并" |

---

### FLOW-005：符文碎片缺少第 4 枚

| 项目 | 内容 |
|------|------|
| **文件** | `src/pages/Clinic.tsx` L324-L361 |
| **描述** | 大纲设计7枚碎片但代码只实现了6枚。碎片#4（应在赵启离职照片的电脑屏幕区域）缺失。结局C需要 `fragments.length === 7` 才能触发。 |
| **影响** | 隐藏结局C在当前代码中**永远无法触发** |
| **修复方案** | 在 `showImageModal` 的照片区域（电脑屏幕右上角）添加第4枚碎片的隐藏交互 |

---

### FLOW-006：碎片收集无去重保护

| 项目 | 内容 |
|------|------|
| **文件** | `src/GameContext.tsx` |
| **描述** | `addFragment` 函数直接往数组 push 碎片编号，没有检查是否已存在。玩家可以反复点击同一个隐藏碎片，使 `fragments.length` 超过7。 |
| **影响** | 可能导致碎片数量显示异常；但不会阻止结局C |
| **修复方案** | `addFragment` 内增加 `if (!fragments.includes(id)) { ... }` 去重检查 |

---

### FLOW-007：桌面缺少「快捷方式」浏览器入口

| 项目 | 内容 |
|------|------|
| **文件** | `src/pages/Desktop.tsx` |
| **描述** | 浏览器图标距离其他文件较远且样式不够突出，新玩家可能注意不到它是推进剧情的关键。 |
| **影响** | 第一幕到第二幕的过渡不流畅 |
| **修复方案** | 将浏览器图标做成桌面上最显眼的快捷方式样式 |

## 四、体验级问题（P2）

### UX-001：完全没有音效

| 项目 | 内容 |
|------|------|
| **描述** | 整个项目没有任何音频相关代码。恐怖游戏中音效是氛围的生命线。 |
| **建议** | 官网：轻柔BGM；OA：低频嗡鸣；终局：防空警报+木鱼混合；发现线索时的音效提示 |

### UX-002：缺少渐进式 UI 异变效果

| 项目 | 内容 |
|------|------|
| **描述** | 大纲中的恐怖UI细节几乎全未实现：❌"安宁"闪烁变"安息"、❌ 专家照片闪帧、❌ 论坛时间显示"丑时三刻"、❌ OA滚动条变红烛、❌ 伪监控弹窗、❌ 关闭拦截 |

### UX-003：论坛帖子数量太少

| 项目 | 内容 |
|------|------|
| **文件** | `src/pages/Forum.tsx` |
| **描述** | 只有4个帖子，关键线索帖子一目了然，失去搜索的乐趣。建议增加到12+个。 |

### UX-004：浏览器缺少前进/后退功能

| 项目 | 内容 |
|------|------|
| **文件** | `src/components/Browser.tsx` |
| **描述** | 前进/后退按钮为纯装饰（`cursor-not-allowed`），玩家切换页面后无法回退。 |

### UX-005：官网首页内容太单薄

| 项目 | 内容 |
|------|------|
| **文件** | `src/pages/Clinic.tsx` L158-L167 |
| **描述** | 首页只有一段文字。应该像真实医疗机构官网一样有轮播Banner、专家卡片、好评展示等。 |
