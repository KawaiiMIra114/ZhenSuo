## 🟣 DESIGN — 用户提出的关键设计问题

以下三个问题由用户在审查过程中提出，属于**产品设计层面的核心问题**，不在原缺陷报告范围内但优先级极高。

### DESIGN-001：桌面使用率过低 & 设计简陋

| 项目 | 内容 |
|------|------|
| **问题描述** | 1. 桌面作为游戏的"主基地"使用率极低——玩家看完三个文件后就再也不回来了。2. 背景使用了一张"木桌"纹理，但游戏设定是操作笔记本电脑，视觉与逻辑不一致。3. 桌面整体设计简陋，缺乏操作系统的沉浸感。 |
| **涉及文件** | `src/pages/Desktop.tsx` |
| **根本原因** | 桌面被设计为纯过渡页面而非交互枢纽。缺少操作系统式UI元素（任务栏、时钟、壁纸、通知、回收站等）。 |
| **修复方案** | **① UI重设计：** 将背景从木桌改为笔记本电脑屏幕/深色操作系统界面；增加任务栏（时钟、WiFi图标、电量）；增加桌面壁纸（可以是妹妹照片或默认系统壁纸）。**② 功能扩展：** 增加"邮件客户端"应用（剧情邮件在此显示）；增加"文件管理器"（整理已收集的线索文件）；桌面在剧情推进中可出现新图标/通知（如"收到新邮件"弹窗）。**③ 回访激励：** 关键线索触发后桌面出现新内容（新邮件、新文件、系统异常通知），让玩家有理由反复回到桌面。 |
| **工作量** | 2-4小时 |
| **优先级** | **高** — 直接影响游戏核心体验 |

---

### DESIGN-003：世界观不清晰，沉浸感缺失

| 项目 | 内容 |
|------|------|
| **问题描述** | 1. 世界观设定散布在各页面中，缺少系统性呈现——玩家无法建立"安宁深眠诊所"的完整认知。2. 玩家能获取到的信息太少，整个流程更像是"为了解密而解密"。3. 角色动机模糊（主角为什么要调查？妹妹是什么人？诊所的真实目的？）难以引发情感共鸣。4. 缺少环境叙事——没有日记、留言、监控录像等"发现式"叙事元素来丰富故事。 |
| **涉及文件** | `Clinic.tsx`、`Forum.tsx`、`OA.tsx`、`Desktop.tsx`（全局叙事问题） |
| **根本原因** | 游戏在开发时侧重解谜机制搭建，叙事层被当作"填充内容"而非核心支柱。世界观文档（v2.0）虽然详尽，但代码中实际呈现的叙事密度远低于设计。 |
| **修复方案** | **① 信息密度提升：** 官网增加更多"表面正常、细看不对劲"的内容（患者感言暗藏求救信号、专家介绍藏玄机、新闻报道自相矛盾）。论坛增加更多"幸存者"和"家属"的帖子，通过不同视角拼凑真相。OA系统增加员工聊天记录/内部邮件，展示内部人员的恐惧与挣扎。**② 环境叙事增加：** 桌面增加"妹妹的最后短信"、"爸爸的语音留言"等情感锚点。浏览器历史记录中可以发现妹妹之前搜索过的内容。增加一个"监控截图"页面展示诊所走廊的异常画面。**③ 角色塑造：** 通过序章（已完成Prologue）、桌面照片、聊天记录等方式让玩家了解主角和妹妹的关系。在论坛中加入妹妹入院前发过的帖子。**④ 叙事节奏：** 在每个解谜节点后穿插叙事奖励——解开一个谜题后不仅获得下一步线索，还获得关于真相的一块碎片。 |
| **工作量** | 4-6小时（需要大量文案撰写） |
| **优先级** | **最高** — 这是游戏的灵魂问题 |
