# 《安宁深眠诊所：拘魂网》 游戏流程与前端解谜闭环合集（V2.0 终极整合版）

> **统筹起草：** Agent
> **定位：** 本文档是所有前端开发、组件路由、线索链排布、乃至 `context.tsx` 状态机记录的**绝对技术与流程指南**。
> **原则声明：** 本文档内容严密遵循 Part 1 世界观法则。每一把所谓的“钥匙（密码、链接）”必须且只能经由游戏内部**合乎逻辑的图文信息导向**，不含任何空中楼阁式的解谜。

## 一、 系统全景与技术落地边界

### 1.1 核心可信载体（操作主视场）
游戏必须伪装成一个单页的、可以自由乱点的现代浏览器应用。我们切分四个主载域：
- **桌面视窗 (`Desktop.tsx`)**：主角唯一的“真实”操作点。包含收件箱、照片扫描件阅读器、日历组件。
- **白区·深眠官网 (`Clinic.tsx`)**：代表秩序。通过文章伪装谎言。包含：动态（公告中心）、专栏、挂号复诊入口。
- **灰区·病友论坛 (`Forum.tsx`)**：代表无序的真实。各种吐槽、抱怨、传销式夸赞混杂其中。
- **黑区·内部系统 (`OASystem.tsx`)**：赤裸裸的罪恶阵法中控台。复古命令行风，只看密码不看人。

### 1.2 何为“有迹可循”的强闭环？
在构建谜题时，必须以**“触发器 -> 行为导向 -> 交叉验证 -> 获得结果”**这一四部曲来写代码设限。
*反面案例*：某张纸写着“你去官网搜3月14号”。
*正面案例（本游规范）*：在邮箱的一角发现日期“03-14”有红色加粗下划线异常，同时发现当月的日历控件上3月14号被画了黑叉。（没有任何提示字眼，玩家自然会觉得这日子不干净，然后去带搜索框的地方查）。

## 二、 全流程解谜链路纪要与推演

这里记录了从玩家进入网页的第一秒，到看到通关界面的每一步操作链。开发者需按照此主线设计组件间的跳转逻辑。

### 阶段一：打破认知隔阂（10-15 分钟）
**【终极目标】**：找到并认清深眠官网在撒谎，顺藤摸瓜找到通向内部系统（OA）的大门。

- **触发器：** 玩家进入应用，桌面上有封名为“关于林晓（急件）”的加密邮件。点开只有一句话：“用她挂号条上的码查进度，别信官方通告。”
- **行为导向：** 玩家点击桌面另一份文件【林晓门诊单扫描件】，翻转背面（图片点击翻转特效），获取条码：`LX-044-YIN`。
- **交叉验证（找门）：** 玩家打开官网，在【挂号及档案查询】一栏输入此码。
  - *系统反馈弹窗*：“您检索的档案已转入【特殊残破资源】库，详细交割处理通告请参阅我院最新发表的医务管理声明。”
- **获得结果（锁定大门与钥匙孔）：** 玩家带着疑问，去【公告中心】翻阅近期的所有声明。其中有一篇《关于某患者转院清算的特别声明》，文本有两处**粗体异常标记**（挂载 `InvestigateNode`）：
  - “此类患者出现机房幻视事件，将由**前台值班经理（工号 8023）**全权进行家属信息切断处理。”
  - “该次事故责任人 IT 部赵某已吊销系统底层访问权。”
- **大门初现：** 在【员工风采】专区翻看一篇关于“赵某离职”的表面官样文章，赵某配图中身后的电脑屏幕，高亮显示着内网入口：`oa.tranquil-sleep.com/login`。
- *阶段结算：* 玩家手中握住了门（OA网址）和锁眼（账号 8023）。

### 阶段二：渗透暗面网络（密码拼图）（15-20 分钟）
**【终极目标】**：利用前台经理的特殊习惯，推理出 8023 账号的密码并登入系统。

- **触发器：** 官网上的员工守则公告（或者是底部带星号的极其不要脸的《信息安全免责条款》）提到：“为防窃取，所有员工临时密码重置标准为：**个人日常口头禅/口令的纯拼音首字母缩写**”。
- **行为导向：** 玩家需要知道这名 8023 号经理的口头禅。自然而去的地方就是供患者灌水的病友群——论坛。
- **交叉验证：** 玩家在论坛顶部搜索框输入“8023”或“前台”。跳出三个患者的吐槽贴。
  - A 患者贴：“那个前台经理好变态，上次送我去做 DNR 之前，一直低着头在我耳边叨叨什么‘浮生无量’。”
  - B 患者跟帖纠正：“你听力有问题吧姐们，那是道观里的说法，叫‘福生无量天尊’。”
- **获得结果：** 拼音缩写 `fswltz`。玩家返回 OA 界面，输入 `8023 / fswltz`。系统界面撕裂一下（伪造的一层绿屏黑底 UI 载入），密码攻破。

### 阶段三：深渊抽丝剥茧（机密拼图）（20-30 分钟）
**【终极目标】**：挖掘在底层维护的赵启留下的抵抗程序片段（上半与下半代码），并拼接获得太极重光覆盖码。

这是在复古绿字风格的 OA 界面内进行的。
- **找寻前半段：** 
  - 玩家进入【采购入库审批单】。看到了前文设定的海量朱砂和雷击木采购。此时 `InvestigateNode` 侦测玩家查阅完毕后，解锁赵启生前留下的一张被权限封锁的【巡检自证照片】。
  - 玩家查阅该机房照片，发现7号水冷机柜侧板贴着巨大黄符。点击符纸（撕纸音效+图片图层剥落），露出写在铁皮上的血字：`下半段阵法覆写函数：JiuKu`。
- **找寻后半段：**
  - 赵启的遗留记事本写道：“林医生是个怂包。但只有她能在对外发的科普专栏里夹带剩下的那一半私货，只有那里能不被内网防火墙刷掉。”
  - 玩家退回现代白底页面的官网【专家专栏】。查阅林医生的六篇文章。发现中间连续的四篇标题，首字连起来极其生硬（参阅 Part 1 的林医生四篇文章名）。
  - 玩家依靠自身推理提取：`太、乙、救、苦`。
- *阶段结论：* 获得最终覆盖码 `TaiYiJiuKu`。

### 阶段四：赛博道场（终末裁决）（15 分钟）
**【终极目标】**：系统崩塌狂欢，作出你的抉择。

- **终极执行：** 玩家用赵启残存的超管账号在 OA 系统底部唤醒了极隐藏的 `[终端核心校准]` 输入框。键入 `TaiYiJiuKu`。
- **视听崩坏：**（这段极度考验前端特效表现）。触发 `onFinalExecution` 流。全屏 DOM 元素开始溶解为暗红色粒子（通过 CSS `backdrop-filter` 和 `canvas` 扰动层）。正常的浏览器标签栏伪 UI 变为繁体咒文。鼠标指针被替换为滴血的香柱。防空警报低频响起。
- **高维对讲：** 在巨大的太岁（莫比乌斯无限符号化作的跳动内脏UI）前，林晓微弱的绿色聊天气泡缓慢蹦出，请求哥哥结束这一切。
- **三向抉择按钮组浮现（根据 `GameContext` 中收集的碎片变量渲染）：**
  - 🔥 **【烈火洗城（格式化）】**：调用物理自毁起火代码。换得全员团灭与现实物理性销毁的伪结局。
  - 🔗 **【上行替代（伥鬼）】**：自己输入身份指令覆盖林晓的节点。退回宁静的官网，但主页团队多了一张目光死寂的“玩家照片”。
  - ⭐ **【七星破阵（净化）】**：且仅当玩家集齐全部网页极小角落里的 7 枚 `InvestigateNode` 隐形挂载碎片时可见。系统逆转运行，播放金属炸裂音效反馈，彻底从代码底层清除太岁高维通道，迎出皆大欢喜的终结。

## 三、 组件层级与状态控制（核心前端规范）

为保证玩家游玩顺畅且进度“锁死”，开发务必遵守以下 React 框架守则：

### 3.1 `GlobalgameStateContext` 状态管理
绝对不能让玩家跨越进度硬解谜语，通过以下全局变量切断违规路径：
```typescript
interface GameState {
  // 当前处于四大阶段的哪一阶段
  currentPhase: 1 | 2 | 3 | 4;
  // 已阅读的关键伏笔 ID 列表（用于控制 InvestigateNode 激活）
  readHooks: string[];
  // 是否破译前台
  isOALoggedIn: boolean;
  // 收集到的道门碎片（控制隐藏结局 C 按钮是否弹出的唯一凭证）
  collectedRunes: string[]; 
}
```
**防御拦截：** 比如虽然玩家是个天才，一开始就在 OA 界面敲对了 `8023 / fswltz`，**前端强制抛出拦截**：“错误：操作过速。无前置设备授权记录”。必须检测到 `readHooks` 里存在“读取了相关新闻”的标识，才放行 `true` 向后跳转。

### 3.2 `InvestigateNode` 的全局包装
所有的线索埋点均通过组件包装，确保玩家的每一次提取都有明确正反馈：
```tsx
<InvestigateNode 
   hookId="news_8023" // 唯一标示码 
   condition={phase >= 1} // 只有在一阶段后才变为可选中
   feedbackType="bubble" // 点击显示解谜气泡
   feedbackText="这位前台主管被特意提名...这是处理林晓事件的核心负责人。"
   onReadComplete={() => dispatch('READ', 'news_8023')} // 抛出至系统中心，解锁后续操作
>
  <span className="font-bold">前台值班经理（工号 8023）</span>
</InvestigateNode>
```

### 3.3 路由（Router）规范：拒绝真刷新
该游戏在浏览器环境下属于一个 `HashRouter` 管理下的纯净态 SPA。绝对不允许因为点击“链接”引发真正的浏览器重新载入（这会导致那些惊悚的音轨和临时记录的调查点丢失）。所有的跳页（官网内页、跳转 OA），全部通过渲染一张遮罩动画（如老旧电视雪花闪屏 `noise.gif` 持续 0.8s）进行无缝组件更换。
